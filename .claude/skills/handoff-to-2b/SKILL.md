# 交棒给 2B

## 触发条件

遇到以下情况，通过通信桥交给 2B：
- 架构设计/技术选型
- 核心代码实现
- 复杂问题需要深度推理
- 需要高质量/高成功率的输出
- 自身多次尝试失败后

## 操作方法

1. 在 `bridge-to-2b.md` 末尾追加消息，格式：
   ```
   ## Handoff: <任务标题>

   ---

   ## 来自 Reasonix（<时间>）

   ### 背景
   <已有调研/上下文>

   ### 需要你做的事
   <明确的任务描述>

   ### 我的尝试/方案
   <Reasonix 已经做的探索和结论>

   ### 验收标准
   <明确可验证的标准>
   ```

2. 等待 2B 完成后，从桥接文件读取结果

## 不做的事
- 不需要把琐碎事项交过去
- 不需要在 2B 执行过程中修改 CLAUDE.md
- 不需要重复 2B 做过的验证
