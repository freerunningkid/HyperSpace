## 二、TTS 语音系统（核心变化）

### 当前架构（文件桥接方案）
```
我的回复 → bash ttb.bat → 写入 tts_queue.txt
                ↓
        tts_watch.vbs（桌面会话后台）
                ↓
    SAPI5 Xiaoxiao Natural 扬声器 🔊
```

**为什么是这个方案**：MCP voice_speak 从未连接。bash 沙箱封了音频管道——Python SAPI5、VBS、PowerShell 在沙箱内全部无声。经 7 轮迭代确定文件桥接为唯一可靠方案。

### TTS 关键文件
| 文件 | 作用 |
|------|------|
| `tts.bat` | 写入文本到 `tts_queue.txt` |
| `tts_watch.vbs` | 500ms 轮询队列 + SAPI5 朗读 |
| `tts-start.vbs` | 启动 Watcher（双击用） |
| `tts-startup.vbs` | 开机自启版（已安装到 Startup） |
| `scripts/lib/speak.py` | TTS 双引擎（edge-tts + SAPI5，默认 sapi5） |

### 启动方式
- 手动：双击 `tts-start.vbs`
- 开机自启：已安装到 `shell:startup`
- 每次回复：Agent 调用 `bash D:\Reasonix\tts.bat "<文本>"`
