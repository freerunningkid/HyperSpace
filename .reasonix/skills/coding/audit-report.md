# 审计报告生成工作流

## 适用场景

小金东在中铁大桥局审计部工作，需要定期生成审计报告。本 Skill 提供从原始数据到正式报告的全流程模板。

## 工作流

### Step 1：收集原始数据

从以下来源收集审计证据：

| 来源 | 获取方式 |
|------|---------|
| 财务凭证/报表 | Excel/PDF 文件 → `run_skill("document-tools", "read: 文件路径")` |
| 项目进度数据 | 数据库/系统导出 |
| 会议纪要 | 文字/语音记录 |
| 合同/协议 | PDF 扫描件 |

### Step 2：结构化底稿整理

使用以下模板组织审计底稿：

```
## 审计底稿：<项目名称>
- 审计期间：YYYY-MM-DD ~ YYYY-MM-DD
- 审计对象：<部门/项目/合同>
- 审计类别：<财务/合规/绩效>

### 证据清单
| # | 证据名称 | 来源 | 结论 |
|---|---------|------|------|
| 1 | ... | ... | ... |

### 发现问题
- 问题描述：
- 风险等级：[高/中/低]
- 依据条款：
- 整改建议：
```

### Step 3：生成正式报告

报告模板：

```
# 审计报告

**审计单位**：[单位名称]
**审计期间**：[期间]
**报告日期**：[日期]

## 一、审计概况
[背景描述]

## 二、审计发现
### 2.1 [问题类别1]
[发现详情]

### 2.2 [问题类别2]
[发现详情]

## 三、审计结论
[总体评价]

## 四、整改建议
[分条列出]
```

### Step 4：格式输出

- 草稿输出：Markdown（便于 AI 迭代修改）
- 最终输出：Word/PDF（使用 `run_skill("document-tools", "write: ...")`）
